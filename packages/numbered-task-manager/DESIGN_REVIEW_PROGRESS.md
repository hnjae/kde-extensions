## Completed

- P1 model-index shape invariant: `TaskEntryLogic.mjs` now owns explicit `modelIndexState(...)` classification and actionability helpers. Diagnostics and action boundaries consume the named policy, preserving current unknown-shape actionability while no longer treating that case as simply valid.
- P1 disabled or invisible context-menu actions remain executable: `TaskContextMenuLogic.mjs` now routes descriptors with explicit `enabled: false` or `visible: false` to unavailable routes, and `TaskContextMenuActionDispatcher.qml` converts those routes into structured dispatch failures before any launcher, launcher-activity, or task-model effect can run. Missing availability fields remain actionable to preserve existing submenu leaf behavior.
- P1/P2 virtual desktop ownership: `VirtualDesktopLogic.mjs` now owns desktop identity coercion, desktop-list membership, current/remote desktop qualification, and context-menu checked-state primitives. `TaskScopeLogic.mjs` and `TaskContextMenuLogic.mjs` consume the shared owner, and `TaskEntryLogic.mjs` no longer carries virtual-desktop membership policy.
- P1/P2 context-menu task command dispatch: `TaskActionLogic.mjs` now validates context-menu task request names against an explicit supported-method allowlist, and `TaskContextMenuTaskCommandAdapter.qml` executes supported `TasksModel.request*` calls through explicit branches instead of dynamic `taskModel[result.requestMethod]` dispatch. The broader raw `TasksModel` port finding remains open for launcher, sync, activation, move, and launcher-activity boundaries.
- P1 context-menu role snapshot ownership: `TaskContextMenuRoleLogic.mjs` now owns context-menu live role reads and snapshot construction. `TaskContextMenuRoleState.qml` consumes the focused owner directly, while `TaskContextMenuLogic.mjs` keeps compatibility re-exports during migration. Broader context-menu feature-family extraction remains open.
- P1 context-menu launcher-activity ownership: `TaskContextMenuLauncherActivityLogic.mjs` now owns launcher-activity menu state and update descriptors. `TaskContextMenuLogic.mjs` consumes the focused owner as a composer/facade, and `TaskContextMenuLauncherActivityAdapter.qml` imports the focused owner directly. Broader context-menu feature-family extraction remains open.
- P1 context-menu task-activity ownership: `TaskContextMenuTaskActivityLogic.mjs` now owns task-activity menu state and action descriptors. `TaskContextMenuLogic.mjs` consumes the focused owner as a composer/facade and keeps compatibility re-exports during migration. Broader context-menu feature-family extraction remains open.
- P1 context-menu window-action ownership: `TaskContextMenuWindowActionLogic.mjs` now owns window-task action descriptor policy for new instance, move/resize, minimize/maximize, keep above/below, fullscreen/shade/no-border, capture, and close actions. `TaskContextMenuLogic.mjs` consumes the focused owner as a composer/facade and keeps compatibility re-exports during migration. Broader context-menu feature-family extraction remains open for virtual desktop action descriptors, footer actions, and final composition cleanup.
- P1 context-menu pin-action ownership: `TaskContextMenuPinLogic.mjs` now owns Pin/Unpin state, launcher command descriptors, action descriptors, pin section composition, and the menu-facing launcher pin-state snapshot. `TaskContextMenuLogic.mjs` consumes the focused owner as a composer/facade and keeps compatibility re-exports during migration. `TaskContextMenuLauncherState.qml` imports the focused owner directly while launcher-list membership remains in `LauncherListLogic.mjs`.
- P1/P2 context-menu route ownership: `TaskContextMenuRouteLogic.mjs` now owns route classification, route-kind constants, command-kind constants used by route classification, and route-kind predicates. `TaskContextMenuActionDispatcher.qml` consumes that owner directly, while `TaskContextMenuLogic.mjs` keeps compatibility re-exports during migration.
- P1 launcher sync orchestration: `LauncherSyncLogic.mjs` now owns `persistLaunchers`, `applyLauncherList`, and launcher-list reconciliation sequencing through explicit effect ports. `LauncherSyncAdapter.qml` is a wrapper around real QML model/configuration ports, and executable tests cover successful writes, thrown writes, convergence mismatches, retry, clear, and expiry paths. Broader launcher-list domain/sync separation remains open because `LauncherListLogic.mjs` still carries lower-level sync helpers during migration.
